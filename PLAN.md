## Этап 7. AI-ассистент для улучшения промтов

> Источник: раздел 7 PROJECT.md

### 7.1. Назначение

Функция помогает пользователю улучшить черновик промта перед отправкой в несколько моделей.  
Ассистент работает через **одну** выбранную модель (OpenRouter) и возвращает структурированный результат для выбора в UI.

### 7.2. Требования (из PROJECT.md)

1. Принимать исходный текст промта от пользователя.
2. Отправлять его в одну указанную модель через существующий OpenRouter-клиент.
3. Возвращать:
   - улучшенную версию промта;
   - 2–3 варианта переформулировки;
   - при необходимости — адаптацию под тип задачи (код, анализ, креатив).
4. Встроить в UI:
   - кнопку **«Улучшить промт»** рядом с полем ввода;
   - окно/панель с исходным, улучшенным и альтернативными вариантами;
   - кнопку **«Подставить в поле ввода»** для выбранного варианта.

### 7.3. Архитектура

```
main.py
  └── PromptAssistantDialog   # UI: кнопка, окно, подстановка
        └── models.py
              └── improve_prompt(text, task_type) -> PromptImprovement
                    └── network.py
                          └── send_assistant_request(model_row, system, user)
```

**Новые/изменяемые файлы:**

| Файл | Изменение |
|------|-----------|
| `prompt_assistant.py` | **новый** — системный промт, парсинг ответа, dataclass результата |
| `network.py` | функция отправки запроса ассистенту (system + user message) |
| `models.py` | `improve_prompt()`, выбор модели-ассистента из settings |
| `db.py` | настройки по умолчанию: `assistant_model_id`, `assistant_enabled` |
| `main.py` | кнопка «Улучшить промт», диалог, фоновый поток |
| `DATABASE.md` | описание новых ключей settings |

### 7.4. Настройки (таблица `settings`)

| Ключ | Значение по умолчанию | Описание |
|------|----------------------|----------|
| `assistant_model_id` | ID модели OpenRouter из `models` | Какая модель улучшает промты |
| `assistant_task_type` | `general` | Тип задачи: `general`, `code`, `analysis`, `creative` |
| `assistant_enabled` | `1` | Включён ли ассистент |

Модель-ассистент выбирается **одна** (не все активные). Рекомендуется seed-модель:  
`OpenRouter GPT-4o Mini` или отдельная запись `Prompt Assistant`.

### 7.5. Системный промт и формат ответа

Ассистенту отправляется system-сообщение с инструкцией вернуть **строгий JSON**:

```json
{
  "improved": "улучшенный промт",
  "alternatives": [
    "вариант 1",
    "вариант 2",
    "вариант 3"
  ],
  "notes": "краткий комментарий, что изменено"
}
```

**Правила для system-промта:**
- сохранять смысл исходного запроса;
- делать промт конкретнее, структурированнее;
- для `code` — добавлять требования к языку, формату вывода;
- для `analysis` — добавлять критерии, шаги рассуждения;
- для `creative` — добавлять стиль, тон, ограничения;
- не добавлять лишних пояснений вне JSON.

**Парсинг:**  
`prompt_assistant.py` → `parse_assistant_response(text) -> PromptImprovement`  
При ошибке парсинга — показать сырой ответ и сообщение об ошибке.

### 7.6. Сетевой слой (`network.py`)

1. Добавить `send_chat_completion(model_row, messages) -> str`:
   - `messages = [{"role": "system", ...}, {"role": "user", ...}]`;
   - переиспользовать `_send_openrouter` / `_send_openai_compatible`.
2. Добавить `improve_prompt_request(model_row, user_prompt, task_type) -> str`:
   - собрать system-промт по `task_type`;
   - передать исходный текст пользователя как user-сообщение;
   - вернуть текст ответа модели.
3. Логировать запросы ассистента в `logs/chatlist.log`.

### 7.7. Бизнес-логика (`models.py`)

1. Dataclass `PromptImprovement`:
   ```python
   @dataclass
   class PromptImprovement:
       original: str
       improved: str
       alternatives: list[str]
       notes: str = ""
   ```
2. Функция `get_assistant_model() -> dict | None` — модель из settings.
3. Функция `improve_prompt(text, task_type=None) -> tuple[PromptImprovement | None, str | None]`:
   - проверить, что текст не пустой;
   - получить модель-ассистента;
   - вызвать `network.improve_prompt_request`;
   - распарсить JSON;
   - вернуть результат или текст ошибки.
4. Функция `set_assistant_model(model_id: int)` — сохранить в settings.

### 7.8. GUI (`main.py`)

#### 7.8.1. Кнопка в главном окне

- Добавить кнопку **«Улучшить промт»** рядом с «Отправить» в блоке «Промт».
- Кнопка неактивна, если поле ввода пустое или идёт другой запрос.

#### 7.8.2. Диалог `PromptAssistantDialog`

Структура окна:

| Зона | Содержимое |
|------|------------|
| Верх | ComboBox «Тип задачи»: Общий / Код / Анализ / Креатив |
| Слева / блок 1 | **Исходный промт** (read-only, QTextEdit) |
| Центр / блок 2 | **Улучшенный промт** (read-only, выделен) |
| Низ / блок 3 | **Альтернативы** — список (QListWidget) из 2–3 вариантов |
| Подвал | Кнопки: «Подставить улучшенный», «Подставить выбранный», «Закрыть» |

**Сценарий:**
1. Пользователь вводит промт → нажимает «Улучшить промт».
2. Показывается индикатор «Улучшаем промт...» (QThread, как `SendPromptWorker`).
3. Открывается `PromptAssistantDialog` с результатами.
4. Пользователь нажимает «Подставить» → текст попадает в `prompt_edit`.
5. Диалог закрывается, пользователь может нажать «Отправить».

#### 7.8.3. Worker-поток

```python
class ImprovePromptWorker(QThread):
    finished = pyqtSignal(object, object)  # PromptImprovement, error
```

Не блокировать UI во время запроса к API.

### 7.9. Обработка ошибок

| Ситуация | Поведение |
|----------|-----------|
| Пустой промт | QMessageBox: «Введите текст промта» |
| Нет модели-ассистента | Предложить выбрать в настройках / использовать OpenRouter по умолчанию |
| Нет API-ключа | Сообщение из `network.get_api_key` |
| Ошибка API | Показать текст ошибки в диалоге |
| Невалидный JSON | Показать сырой ответ + «Не удалось разобрать ответ ассистента» |
